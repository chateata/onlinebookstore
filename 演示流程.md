## 🎯 在线书店管理系统演示流程

### **演示总时长**：约25-30分钟
### **演示环境**：Spring Boot + MySQL + 前后端完整系统
### **演示目标**：全面展示系统6大核心功能模块的完整业务流程

---

## **演示步骤详解**

### **步骤1：系统概述与架构介绍** (3分钟)
**操作**：展示项目结构和数据库设计
**讲解文案**：
"大家好！今天为大家演示的是一个完整的在线书店管理系统。这个系统严格按照功能需求规格实现了6大核心功能模块：

1. **供书目录及库存管理** - 完整的书籍信息管理、分类体系和库存控制
2. **采购管理** - 缺书登记、采购单管理和自动库存补充
3. **客户管理** - 用户注册登录、信用等级管理和账户余额系统
4. **顾客订单管理和发货管理** - 完整的订单生命周期管理
5. **供应商管理** - 多供应商体系和供应商书目管理
6. **网上浏览查询** - 多维度书籍查询和客户信息查询

系统采用Spring Boot + MyBatis + MySQL的技术架构，前端使用Thymeleaf模板引擎和Layui组件库，实现了前后端分离的开发模式。数据库设计包含35+张表，支持复杂的业务关系和数据完整性约束。

现在让我带大家从用户视角开始体验这个系统..."

### **步骤2：用户注册与登录** (2分钟)
**操作**：访问首页 → 点击注册 → 填写信息 → 登录
**讲解文案**：
"首先我们从普通用户的视角开始。系统支持完整的用户管理体系：

- **注册功能**：用户可以填写用户名、密码、邮箱等信息进行注册
- **登录验证**：系统会对用户凭据进行验证，成功后建立session会话
- **权限控制**：登录后用户可以访问个人中心、购物车、订单等功能

可以看到系统采用了分层架构设计，Controller层处理请求，Service层封装业务逻辑，DAO层通过MyBatis操作数据库。这种设计保证了代码的可维护性和扩展性。

注册完成后，系统会自动为用户分配初始信用等级（三级客户），并初始化账户余额为100。"

### **步骤3：书籍浏览与查询功能** (4分钟)
**操作**：浏览首页 → 按分类筛选 → 搜索书籍 → 查看详情
**讲解文案**：
"现在我们来体验系统的网上浏览查询功能，这是系统的核心功能之一：

**多维度查询支持**：
- **按书名查询**：支持模糊匹配，输入'三体'可以找到相关书籍
- **按作者查询**：系统支持多作者关联，可以按第一、第二作者等进行查询
- **按出版社查询**：精确匹配出版社名称
- **按关键字查询**：支持多个关键字的组合查询
- **按ISBN查询**：精确匹配国际标准书号

**书籍信息展示**：
- 完整的书籍元数据：ISBN、书名、作者、出版社、出版日期、价格等
- 库存状态实时显示
- 支持丛书系列展示
- 书籍封面和详细描述

这个查询系统采用了MySQL全文索引和优化SQL查询，确保在大规模数据下的查询性能。可以看到系统支持35个书籍分类，从小说到管理类应有尽有。"

### **步骤4：购物车功能体验** (3分钟)
**操作**：添加书籍到购物车 → 查看购物车 → 修改数量 → 删除商品
**讲解文案**：
"购物车是电商系统的核心功能，我们的系统实现了完整的购物车管理：

**购物车特性**：
- **多商品管理**：支持同时添加多本不同书籍
- **数量控制**：实时计算小计和总价
- **库存检查**：添加时自动检查库存是否充足
- **数据持久化**：购物车数据存储在数据库中，支持跨会话保持

可以看到系统采用了shopping_cart表来存储购物车数据，通过外键关联用户和书籍。这种设计既保证了数据完整性，又支持高效的查询操作。

当用户添加书籍到购物车时，系统会检查库存是否充足，如果库存不足会给出相应提示。"

### **步骤5：订单创建与管理** (4分钟)
**操作**：从购物车下单 → 填写收货信息 → 提交订单 → 查看订单详情
**讲解文案**：
"现在我们来体验完整的订单管理流程，这是系统最复杂的业务模块之一：

**订单创建流程**：
1. **订单生成**：从购物车批量创建订单，支持多本书同时下单
2. **信息填写**：收货人、地址、联系方式等配送信息
3. **金额计算**：根据用户信用等级自动应用折扣
4. **库存扣减**：订单确认后自动扣减库存

**订单状态管理**：
- **待审核**：新创建的订单需要管理员审核
- **待付款**：审核通过后等待用户付款
- **待发货**：付款完成后进入发货流程
- **已发货**：订单已发出，等待用户确认收货

系统采用了订单主表(order)和订单明细表(order_item)的设计，支持一个订单包含多本书籍。数据库中还包含了发货状态跟踪字段，确保整个订单生命周期的可追溯性。

特别值得一提的是，系统实现了自动库存扣减机制：当订单创建时，系统会检查库存是否充足，如果库存不足会自动生成缺书记录，并触发采购流程。"

### **步骤6：管理员后台登录** (2分钟)
**操作**：访问后台登录页 → 输入管理员账号 → 进入管理界面
**讲解文案**：
"现在我们切换到管理员视角，体验系统的后台管理功能：

系统采用了RBAC权限模型，管理员账号是预设的（admin/123456），登录后可以访问所有管理功能。后台管理界面采用Layui组件库构建，提供了良好的用户体验。

管理员权限包括：
- 用户管理：查看和修改用户信息
- 书籍管理：增删改查书籍信息
- 订单管理：审核订单、修改发货状态
- 供应商管理：管理供应商信息和供货关系
- 采购管理：处理缺书记录和采购单
- 库存管理：实时查看和调整库存

这种设计确保了前后台职责分离，普通用户只能访问前端功能，而管理员拥有完整的系统控制权。"

### **步骤7：书籍库存管理** (3分钟)
**操作**：查看书籍列表 → 添加新书 → 编辑现有书籍 → 设置最小库存
**讲解文案**：
"书籍库存管理是书店日常运营的核心功能，我们的系统实现了完整的库存管理体系：

**库存管理特性**：
- **实时库存显示**：每本书的当前库存数量一目了然
- **最小库存设置**：为每本书设置最低库存阈值
- **自动缺书提醒**：当库存低于最低值时自动生成缺书记录
- **库存位置管理**：支持库存存放位置的可视化展示

可以看到系统为每本书都设置了min_stock字段，当库存低于这个值时，数据库触发器会自动插入一条shortage记录。这种自动化处理大大提高了运营效率。

此外，系统还支持丛书管理，一本书可以属于某个系列，这对于管理套装书和系列图书非常重要。"

### **步骤8：供应商管理体系** (4分钟)
**操作**：查看供应商列表 → 添加新供应商 → 管理供应商书目 → 查看供货价格
**讲解文案**：
"供应商管理是采购体系的基础，我们的系统实现了多供应商的完整管理体系：

**供应商管理功能**：
- **基本信息管理**：供应商名称、联系方式、地址等
- **供应商书目管理**：每个供应商维护自己的书目清单
- **供货价格管理**：不同供应商的同本书可以有不同价格
- **状态控制**：供应商和书目的启用/停用状态

系统采用了supplier表和supplier_book表的设计，supplier_book表存储了供应商发布的书籍信息，包括ISBN、书名、定价、供货价等。这种设计支持一个书店从多个供应商采购同一本书，选择最优价格。

特别的是，系统还支持丛书系列管理，可以将相关书籍归类到同一个系列中，便于批量采购和管理。"

### **步骤9：采购管理流程** (4分钟)
**操作**：查看缺书记录 → 生成采购单 → 确认到货 → 库存增加
**讲解文案**：
"采购管理是连接库存和供应商的关键环节，我们的系统实现了完整的采购业务流程：

**采购管理流程**：
1. **缺书记录生成**：
   - 订单扣减库存时自动生成
   - 库存低于最低值时自动生成
   - 支持手动登记缺书

2. **采购单生成**：
   - 根据缺书记录选择供应商
   - 自动计算采购数量和金额
   - 支持多供应商的采购分配

3. **到货处理**：
   - 确认采购单到货
   - 自动增加库存数量
   - 标记缺书记录为已处理

系统采用了shortage表存储缺书记录，purchase_order表和purchase_order_item表管理采购单。这种设计确保了采购流程的可追溯性和数据完整性。

数据库触发器还实现了自动库存检查，当库存不足时会自动生成采购需求，大大提高了运营效率。"

### **步骤10：客户信用等级管理** (3分钟)
**操作**：查看用户列表 → 调整信用等级 → 查看折扣设置 → 模拟消费升级
**讲解文案**：
"客户信用等级管理体现了系统的精细化运营能力：

**信用等级体系**：
- **五级信用体系**：从一级到五级，对应不同的折扣和透支额度
- **自动升级机制**：根据累计消费金额自动提升等级
- **手动调整权限**：管理员可以根据实际情况调整用户等级

**等级权益**：
- 一级：0%折扣，无透支
- 二级：2%折扣，无透支  
- 三级：5%折扣，可透支100元
- 四级：8%折扣，可透支500元
- 五级：10%折扣，可透支1000元

系统通过credit_level表管理等级定义，user_info表中的credit_level_id字段关联用户等级。这种设计既保证了数据的规范性，又支持灵活的等级调整。

可以看到，信用等级系统不仅提供了差异化的服务，还通过透支机制增强了用户的购物体验。"
